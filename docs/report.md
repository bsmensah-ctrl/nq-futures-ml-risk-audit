# Machine Learning-Gated NQ Futures Strategy: Walk-Forward Validation, Risk Controls, and Failure Analysis

## Problem

The research problem was predicting and controlling risk in an NQ futures strategy. The strongest version of the project is not "I built a profitable trading bot." The stronger and more truthful claim is that the project built a validation pipeline to test whether a machine-learning volatility gate could reduce overtrading, drawdown, and regime fragility in a historical NQ strategy.

The first portfolio draft had a weak claim: it used a 9+ "Sharpe-like" number that is actually a trade-level t-statistic. That number is not credible as a finance-style Sharpe ratio. The refreshed package uses monthly annualized Sharpe, slippage stress, walk-forward folds, and failure cases instead.

After the overfitting audit, the claim should be narrower again: the best supported result is an NQ-specific historical research pipeline with material execution risk. It is not a broad futures strategy and not a live-trading proof.

## Data

The cleaned portfolio package uses selected evidence from the original `NQ_Project` workspace and a fresh rerun completed on June 5, 2026:

- Databento-derived NQ 1-minute bars used in the Phase 0 validation.
- Fresh Phase 0 run `phase0_20260605T225254Z`, with all four scripts returning `ok`.
- Timestamped trade streams generated by `src/fresh_conservative_validation.py`.
- Independent replication and overfitting audit generated by `src/overfit_replication_audit.py`.

Raw vendor data, model pickle files, API keys, and live broker configuration are intentionally excluded.

## Method

The tested strategy is an ML-gated synthetic straddle. When the gate fires, the backtest enters both long and short exposure around an NQ bar, then exits with asymmetric profit target / stop loss logic. The method relies less on predicting direction and more on identifying bars where one side moves far enough to overcome the losing stopped leg and transaction costs.

Validation components:

- ML gate compared with no-ML and directional baselines.
- 12-month train / 3-month test / 3-month slide walk-forward validation.
- Delayed-entry stress to approximate execution lag.
- Drawdown-aware 25k-account parameter grid.
- Slippage stress from 0.5 to 2.5 NQ points per leg.
- Failure analysis of directional, overtrading, and cross-instrument variants.
- Independent rebuild from raw NQ CSV without importing the original strategy module.
- Clean-eval and purged walk-forward reruns to avoid using the test fold for early stopping.
- Label-shuffle null test, feature ablations, block-bootstrap Monte Carlo, same-bar adverse ordering stress, and cross-instrument sanity checks.

## Results

Fresh rerun headline:

| Variant | Trades | PnL | Win Rate | Finance-Style Metric |
|---|---:|---:|---:|---:|
| Base modeled cost, 0.5 pt/leg | 2,022 | +$142,181 | 69.9% | Monthly Sharpe 6.71 |
| Conservative stress, 1.5 pt/leg | 2,022 | +$61,301 | 66.6% | Monthly Sharpe 2.85 |
| Severe stress, 2.0 pt/leg | 2,022 | +$20,861 | 63.2% | Monthly Sharpe 0.93 |
| Directional filtered comparison | 2,022 | -$6,275 | 28.4% | Monthly Sharpe -0.48 |

The portfolio headline should use the 1.5 pt/leg stress case, not the unstressed or trade-level t-statistic case. It is still strong, but it is no longer unbelievable on its face, and it shows execution sensitivity.

Independent audit results:

- Five walk-forward folds were all positive for the filtered straddle.
- Under 1.5 points of slippage per leg, 13 of 16 months were positive.
- Clean-eval validation, which avoids test-fold early stopping, produced +$60,511 at 1.5 pt/leg with monthly Sharpe 2.91.
- Purged clean-eval validation produced +$56,145 at 1.5 pt/leg with monthly Sharpe 2.87.
- Label-shuffle testing produced zero trades, which supports that the gate is not simply firing after target randomization.
- At 2.5 points of slippage per leg, PnL turned negative, making execution quality the major risk boundary.
- Independent same-logic cross-instrument checks failed on ES and MGC, so broad futures generalization is not supported.
- The 25k-account optimization identified passing parameter sets under explicit drawdown constraints.

## Failure Cases

The failure cases are part of the proof:

- Directional prediction was not strong enough by itself.
- No-ML ATR expansion overtraded and accumulated cost drag.
- Earlier deep-validation logs showed low-volatility and pre-market periods could damage results.
- Cross-instrument audit checks failed on ES and MGC, limiting generality beyond this NQ-specific setup.
- Execution-cost stress shows the strategy is not robust to unlimited slippage.
- Same-bar adverse ordering produced a large loss, so intrabar fill ordering is the most serious unresolved execution assumption.
- The original script used test-fold labels for LightGBM early stopping and shared five boundary timestamps between train/test slices; the clean-eval and purged reruns were added to correct that validation weakness.

## Limitations

The results are historical backtests with modeled execution costs. They are not live fills. Remaining risks include overfitting, stale regime assumptions, intrabar fill ambiguity, broker execution differences, slippage under stress, and survivorship of parameter choices.

## Next Steps

The next defensible step is a locked forward paper-trading period with no parameter changes. After that, the project needs fill-quality comparison against a broker simulator, broader futures-universe testing, stricter raw-data provenance, and daily risk controls before any live deployment claim.
